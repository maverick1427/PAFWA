Part 1 — Create Supabase Database
Supabase is your free cloud database — it stores all inventory, sales, users and receipts
Supabase Setup6 steps · ~15 min
💡

What is Supabase? It's like a free database in the cloud. Your PAFWA data (items, sales, users) will be stored there safely. It's free for small apps and needs no server.

1
Create a Supabase Account
a
Open your browser and go to https://supabase.com
b
Click the big Start your project button
c
Click Sign in with GitHub — it will open GitHub login. Sign in with your GitHub account (or create one free at github.com first)
d
Supabase will ask permission to connect to GitHub. Click Authorize supabase
e
You will land on the Supabase dashboard. You have an account! ✅
2
Create a New Project
a
On the Supabase dashboard, click + New project
b
Under Organisation — keep the default (your username)
c
Under Project name type: pafwa-apf
d
Under Database Password — click Generate a password and copy the generated password to Notepad. You may need it later.
e
Under Region — select Southeast Asia (Singapore) — closest to Pakistan for fastest speed
f
Click Create new project
g
Wait about 2 minutes. You will see a progress bar. When done, you'll see the project dashboard.
3
Copy Your Project URL and Anon Key
⚠️

Very important! You need these two values to connect the app to your database. Save them in Notepad.

a
In your Supabase project, look at the left sidebar and click ⚙ Project Settings (the gear icon at the bottom)
b
Click API in the settings menu
c
Under Project URL — copy the URL that looks like: https://xxxxxxxxxxx.supabase.co and paste in Notepad
d
Under Project API keys, find the row that says anon public — click the Copy icon next to it. Paste this long key in Notepad too.
✅

You should now have 2 things saved in Notepad:
1. Project URL → https://xxx.supabase.co
2. Anon Key → eyJhbGci... (very long text)

4
Enable Email Authentication
a
In the left sidebar click Authentication
b
Click Providers tab at the top
c
Find Email in the list — it should already be enabled (toggle is blue). If not, click to enable it.
d
Make sure "Confirm email" is turned OFF (toggle grey) — this lets users log in immediately without email confirmation
e
Click Save
5
Create the Storage Bucket for Item Images
a
In the left sidebar click Storage
b
Click New bucket
c
In the Name box type: item-images
d
Toggle "Public bucket" to ON (blue) — this allows item images to be visible in the app
e
Click Save
6
Add Your GitHub Pages URL to Allowed Origins
ℹ️

This allows your website (on GitHub Pages) to communicate with Supabase. Without this, the app will be blocked.

a
In Supabase, go to ⚙ Project Settings → API
b
Scroll down to Allowed CORS origins (or "API Settings")
c
Also go to ⚙ Project Settings → Authentication → URL Configuration
d
In Site URL enter: https://maverick1427.github.io/PAFWA
e
In Redirect URLs click Add URL and add: https://maverick1427.github.io/PAFWA
f
Click Save

Part 2 — Run the Database Schema
This creates all the tables and security rules in your Supabase database
Database Schema3 steps · ~5 min
⚠️

Do this only once. This sets up all the database tables for inventory, sales, users, etc. Running it twice is safe — it won't break anything.

1
Open the SQL Editor in Supabase
a
In your Supabase project, look at the left sidebar and click SQL Editor (the </> icon)
b
Click + New query at the top left
c
You will see a big empty text area — this is where you will paste the SQL code
2
Open and Copy the Schema File
a
Open your PAFWA-SB folder on Desktop
b
Find the file called schema.sql
c
Right-click it → Open with → Notepad
d
Press Ctrl + A to select all the text, then Ctrl + C to copy it
3
Paste and Run the Schema
a
Go back to Supabase SQL Editor. Click inside the empty text area.
b
Press Ctrl + A to select all, then Ctrl + V to paste your copied schema
c
Click the green ▶ Run button at the top right (or press Ctrl + Enter)
d
Wait about 10-15 seconds. At the bottom you should see: Success. No rows returned
✅

Success! Your database now has all tables: items, sales, receipts, users, logs, categories and settings. The security rules are also applied.

❌

If you see a red error message — scroll up in this guide to check that you completed Part 1 fully. Common issue: forgetting to enable Email Authentication.

Run These Commands to Upload to GitHub 🚀

Type each command and press Enter. Wait for each to finish before typing the next.

Command Prompt — run in order
:: Set your name and email (once only on this PC)
git config --global user.name "A Khan"
git config --global user.email "your@email.com"

:: Initialize git in the PAFWA-SB folder

:: Stage all files
git add .

:: Create first commit
git commit -m "PAFWA v2.0 Supabase + GitHub Pages"

:: Set branch name
git branch -M main

:: Connect to your GitHub repo

:: Push (upload) — enter token when asked for password

Part 6 — Troubleshooting
Solutions to every common problem you might encounter
!
"Invalid URL" error when connecting Supabase
Make sure the URL ends with .supabase.co with no trailing slash. Copy it exactly from Supabase Project Settings → API.
!
"Row level security policy" error after login
The SQL schema was not run fully. Go to Supabase SQL Editor, paste the schema.sql content again and run it.
!
GitHub Action fails with red ✗
Click the red ✗ → click the failed job → read the error message. Usually it means GitHub Pages is not enabled. Go to Settings → Pages → set Source to "GitHub Actions".
!
App URL shows 404 page not found
GitHub Pages might not be enabled or the Action hasn't run yet. Wait 2 minutes and refresh. Also make sure the repo is Public (Settings → Danger Zone → Change to public).
!
"Authentication failed" when pushing from CMD
You used your GitHub password instead of the Personal Access Token. Generate a new token at github.com/settings/tokens and use that as the password.
!
Images not uploading to items
The item-images storage bucket was not created or is not public. Go to Supabase → Storage → check if bucket exists with "Public" status.
!
Login works but data doesn't appear
The schema SQL was not run. Go to Supabase → Table Editor — if you see no tables listed, run the schema.sql again in SQL Editor.
!
Staff on another device can't log in
They need to enter the Supabase config on their device first (click "Change Supabase Config" on login page). After that it's saved in their browser.
!
"git is not recognized" in Command Prompt
Git is not installed properly. Go back to Part 3 Step 1, reinstall Git, restart your PC, then try again.
